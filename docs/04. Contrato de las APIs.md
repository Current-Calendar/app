# Contrato de las APIs

## Propósito

El presente documento tiene la intención de informar a los desarrolladores de Current sobre la arquitectura de API REST del proyecto. A continuación se encuentran las rutas a emplear separadas por módulo.

> **Nota:** Todas las rutas comienzan desde la raíz del servidor (p. ej. `http://localhost:8000`). Los errores estándar devuelven `{ "errors": ["..."] }` o `{ "error": "..." }` con el código HTTP correspondiente.

---

## Gestión de usuarios

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/auth/registro/` | Registro de nuevo usuario | `201` `{ "message": "Usuario registrado exitosamente", "usuario": { id, username, email, ... } }` |
| `POST` | `/api/v1/auth/login/` | Inicio de sesión | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/logout/` | Cierre de sesión | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/password_reset/` | Solicitar email de restablecimiento | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/password_reset/confirm/` | Cambiar contraseña con el token del email | _Pendiente de implementar_ |
| `PUT` | `/api/v1/users/me` | Editar perfil propio | `200` Datos del usuario actualizados (UserSerializer) |
| `DELETE` | `/api/v1/users/me` | Eliminar cuenta propia | `202` `{ "message": "Usuario eliminado satisfactoriamente" }` |
| `GET` | `/api/v1/usuarios?search=<query>` | Buscar usuarios por username, email o pronombres | `200` `[ { id, username, email, pronombres, biografia, foto, total_seguidores, total_seguidos, total_calendarios_seguidos }, ... ]` |
| `POST` | `/api/v1/users/{id}/follow/` | Seguir / dejar de seguir a un usuario | `200` `{ "user": <id>, "followed": <bool> }` |
| `POST` | `/api/v1/users/{id}/block/` | Bloquear a un usuario | _Pendiente de implementar_ |

**Body de `/api/v1/auth/registro/`:**
```json
{
  "username": "string",
  "email": "string",
  "password": "string",
  "password2": "string"
}
```

## Gestión de calendarios

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/calendarios` | Crear un nuevo calendario | `201` `{ id, origen, id_externo, nombre, descripcion, estado, creador_id, fecha_creacion }` |
| `GET` | `/api/v1/calendarios/list` | Listar calendarios (con filtros opcionales `?q=<nombre>&estado=<PRIVADO\|AMIGOS\|PUBLICO>`) | `200` `[ { id, nombre, descripcion, estado, origen, creador_id, creador_username, fecha_creacion }, ... ]` |
| `PUT` | `/api/v1/calendarios/{id}/publicar` | Publicar un calendario (cambia estado a PUBLICO) | `200` `{ id, nombre, descripcion, estado, origen, creador, fecha_creacion }` |
| `PUT` / `PATCH` | `/api/v1/calendarios/{id}/editar/` | Editar nombre, descripción o estado de un calendario | `200` `{ id, nombre, descripcion, estado }` |
| `DELETE` | `/api/v1/calendarios/{id}/eliminar/` | Eliminar un calendario (solo el creador) | `204` Sin contenido |

**Body de `POST /api/v1/calendarios`:**
```json
{
  "creador_id": "<int>",
  "nombre": "string",
  "descripcion": "string (opcional)",
  "estado": "PRIVADO | AMIGOS | PUBLICO (opcional, default: PRIVADO)",
  "origen": "CURRENT | GOOGLE | APPLE (opcional, default: CURRENT)",
  "id_externo": "string (opcional)"
}
```

**Body de `PUT/PATCH /api/v1/calendarios/{id}/editar/`:**
```json
{
  "nombre": "string (opcional)",
  "descripcion": "string (opcional)",
  "estado": "PRIVADO | AMIGOS | PUBLICO (opcional)"
}
```

## Gestión de eventos

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/eventos` | Crear un nuevo evento | `201` `{ id, titulo, descripcion, nombre_lugar, fecha, hora, recurrencia, id_externo, calendarios: [int], fecha_creacion }` |
| `PUT` | `/api/v1/eventos/{id}` | Editar un evento existente | `200` `{ id, titulo, descripcion, nombre_lugar, fecha, hora, recurrencia, id_externo, calendarios: [int], fecha_creacion }` |
| `GET` / `DELETE` | `/api/v1/eventos/{id}` | Detalle / borrado de evento | _Pendiente de implementar_ |
| `POST` | `/api/eventos/asignar/` | Asignar un evento a un calendario | `200` `{ "mensaje": "Evento '<titulo>' asignado al calendario '<nombre>'" }` |
| `DELETE` | `/api/eventos/desasignar/` | Desasignar un evento de un calendario | `200` `{ "mensaje": "Evento '<titulo>' desasignado del calendario '<nombre>'" }` |

**Body de `POST /api/v1/eventos`:**
```json
{
  "titulo": "string",
  "fecha": "YYYY-MM-DD",
  "hora": "HH:MM:SS",
  "calendarios": [<calendario_id>, ...],
  "creador_id": "<int>",
  "descripcion": "string (opcional)",
  "nombre_lugar": "string (opcional)",
  "latitud": "<float> (opcional)",
  "longitud": "<float> (opcional)",
  "recurrencia": "string (opcional)",
  "id_externo": "string (opcional)"
}
```

**Body de `POST /api/eventos/asignar/` y `DELETE /api/eventos/desasignar/`:**
```json
{
  "evento_id": "<int>",
  "calendario_id": "<int>"
}
```

## Importación / Exportación de calendarios

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `GET` | `/api/v1/google-auth` | Iniciar flujo OAuth2 de Google Calendar | `302` Redirección a Google OAuth |
| `GET` / `POST` | `/api/calendars/import-google-calendar` | Importar eventos desde Google Calendar (requiere sesión OAuth2 activa) | `200` `{ "message": "Eventos importados exitosamente", "count": <int> }` |
| `POST` | `/api/calendars/import-ios-calendar` | Importar calendario desde iOS Calendar vía URL webcal | `200` `{ "message": "Calendario iOS importado exitosamente", "count": <int> }` |
| `POST` | `/api/calendars/import-ics` | Importar calendario desde archivo `.ics` subido | `200` `{ "message": "Archivo ICS importado exitosamente" }` |
| `GET` | `/api/calendars/{id}/export` | Exportar un calendario a formato `.ics` | `200` Archivo adjunto `text/calendar` |

**Body de `POST /api/calendars/import-ios-calendar`:**
```json
{
  "webcal_url": "webcal://... o https://...",
  "user": "<int>",
  "estado": "PRIVADO | AMIGOS | PUBLICO (opcional, default: PRIVADO)"
}
```

**Body de `POST /api/calendars/import-ics` (multipart/form-data):**
```
file: <archivo .ics>
user: <int>
estado: PRIVADO | AMIGOS | PUBLICO (opcional, default: PRIVADO)
```
